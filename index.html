Learning Unit 1: HTML und Werkzeuge

Auszeichnungssprachen:
Maschinenlesbare Sprache, Gliederung und Formatierung, keine Programmiersprache.

HTML:
HyperText Markup Language, Standard für Webseiten, aktuelle Version: HTML5.

Element:
Bausteine von HTML, markiert durch Tags, können verschachtelt sein.

Tag:
Schlüsselwort, markiert Elemente, geschrieben in spitzen Klammern, z.B. <body>.

Attribute:
Zusätzliche Angaben im Tag, z.B. lang="de", bestehen aus Schlüsselwort und Wert.

CSS:
Cascading Style Sheets, verantwortlich für Layout und Darstellung.

HTML Grundgerüst:
Vorlage in Webstorm:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Title</title>
  </head>
  <body>

  </body>
</html>

<!DOCTYPE html>
Legt Dokumentenart fest, kein schließendes Tag.

<html> ... </html>
Ganzes HTML-Dokument muss zwischen diesen Tags stehen, lang=?? angeben.

<head> ... </head>
Informationen über Seiteninhalt, nicht angezeigt im Browser, zwingend: <title> und <meta charset="utf-8">.

<body> ... </body>
Inhalt der Webseite, angezeigt im Browser.

Learning Unit 2: Aufbau HTML-Datei

Block- und Inline-Elemente:

2 Arten von HTML-Elementen:

Block-Elemente

  • Absätze: <p>
  • Listen: <ul>, <ol>
  • Überschriften: <h1> bis <h6>
  • Artikel: <article>
  • Abschnitte: <section>

Inline-Elemente

  • Links: <a>
  • Hervorgehobene Wörter: <em>
  • Wichtige Wörter: <strong>
  • Kurze Zitate: <q>

Cascading Style Sheets (CSS)

Aufbau eines Selektors in CSS:

selector {
  property: value;
}

Deklaration eines Selektors:

p {
  color: red;
  font-size: 14px;
}

Kommentare für CSS-Code verwenden:

/* Das ist ein Kommentar in CSS */
p {
  color: blue; /* Dieser Kommentar ist inline */
}

Formatierung des Quellcodes:

body {
  background-color: #f0f0f0;
  margin: 0;
  padding: 0;
}

Learning Unit 4: Codestyle und Statische Code Analyse

4a. Codestyle

  • Dateinamen in Kleinbuchstaben schreiben
  • Elementnamen in Kleinbuchstaben schreiben
  • Kleinbuchstaben für Attributnamen verwenden
  • Attributwerte immer in Hochkomma setzen
  • Alle HTML-Elemente schließen
  • alt, width und height für Bilder angeben
  • Leerzeichen und Gleichheitszeichen korrekt verwenden

Good:

<link rel="stylesheet" href="styles.css">

Bad:

<link rel = "stylesheet" href = "styles.css">
  • Vermeiden Sie lange Codezeilen
  • Leerzeilen und Einrückungen korrekt verwenden
  • <title>-Element immer angeben
  • HTML-Kommentare verwenden

Beispiel für HTML-Kommentare

<!-- Das ist ein HTML-Kommentar -->

4b. Statische Code Analyse

  • Automatische Formatierung
  • Code-Style-Konfiguration
  • Linters und Code-Analyse-Tools
  • Live-Vorlagen und Code-Snippets
  • Code-Inspektionen
  • Linter für HTML installieren

Linter: Programm zur Quellcode-Analyse, identifiziert Probleme, Stilprobleme und Verstöße gegen Best Practices. Unterstützt sauberen und konsistenten Code, verbessert die Codequalität. Beispiel: htmlhint auf Notebook installieren und verwenden.

Beispiel für die Verwendung von htmlhint

# Installieren Sie htmlhint
npm install -g htmlhint

# Verwenden Sie htmlhint zur Analyse Ihres HTML-Codes
htmlhint IhreDatei.html

Learning Unit 5: HTTP und Adressen

HTTP-Methoden

  • GET: Daten abrufen
  • POST: Daten senden
  • PUT: Daten aktualisieren
  • DELETE: Daten löschen

HTTP Statuscodes

  • 200: OK
  • 404: Nicht gefunden
  • 500: Serverfehler

Domain Name System (DNS)

Namensauflösung: Konvertiert Domainnamen in IP-Adressen

Adressen im Web

Absolute Adressen

Vollständige URL, z.B. https://example.com/path

Relative Adressen

  • Nur Dateiname: props01.png (gleicher Ordner)
  • Mit Punkt: ./img/props01.png (aktuelles Verzeichnis)
  • Mit Schrägstrich: /img/props01.png (Wurzelverzeichnis)

Learning Unit 6: Formularelemente

Formular-Elemente

  • <form>: Formularcontainer
  • action: Ziel-URL
  • method: Übertragungsmethode (GET, POST)

Input-Elemente

  • <input>: Eingabefeld
  • name: Name des Eingabefelds
  • type: Typ des Eingabefelds (text, password, email, etc.)
  • Sonstige Attribute:
    • minlength: Minimale Zeichenanzahl
    • required: Pflichtfeld
    • max: Maximaler Wert
  • <textarea>: Mehrzeiliges Eingabefeld
  • <label>: Beschriftung für Eingabefelder
  • Button-Typen:
    • type="submit": Senden
    • type="reset": Zurücksetzen
    • type="button": Standard-Button

Validierung

Attribut Beschreibung types
required Pflichtfeld alle außer image, button
minlength / maxlength Minimale und maximale Zeichenanzahl text, email, search, password, tel, url
min / max Minimaler und maximaler Wert number, date, time

Learning Unit 8: Seitennavigation

Wichtige Attribute des <a>-Tags

  • href: Ziel-URL
  • target: Öffnen des Links in einem neuen Fenster oder Tab
  • title: Tooltip-Text für den Link

Grundstruktur des <nav>-Elements

Das <nav>-Element dient der strukturellen Auszeichnung von Navigationsbereichen.

Anwendung des <nav>-Elements

Um die Navigation eines Dokuments zu gruppieren und zu kennzeichnen.

Navigation

  • Navigation über mehrere Seiten mit dem <a>-Tag
  • Navigation auf einer Seite mit Sprungmarken

Grundlagen der Sprungmarken

Mit Sprungmarken können Benutzer direkt zu einem bestimmten Abschnitt innerhalb einer Seite springen.
